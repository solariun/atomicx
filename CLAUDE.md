# Project Premises

## Language
- All documentation, comments, and commit messages in English

## Commit Convention
- Do not use "Author" or "Co-Authored-By" in commits
- Use "Contributor: Claude" when attribution is needed

## Code Style
- Linux kernel coding standard

## Tone
- Neutral — no hype, no fluff, no emojis

## Target Audience
- Embedded engineers and curious people of all levels, from basic to advanced
- Documentation should be accessible but not dumbed down
