\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md30}{}\doxysection{Configuration keys}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md30}

\begin{DoxyItemize}
\item {\ttfamily board\+\_\+manager}
\begin{DoxyItemize}
\item {\ttfamily additional\+\_\+urls} -\/ the URLs to any additional Boards Manager package index files needed for your boards platforms.
\end{DoxyItemize}
\item {\ttfamily daemon} -\/ options related to running Arduino CLI as a \href{https://grpc.io}{\texttt{ g\+RPC}} server.
\begin{DoxyItemize}
\item {\ttfamily port} -\/ TCP port used for g\+RPC client connections.
\end{DoxyItemize}
\item {\ttfamily directories} -\/ directories used by Arduino CLI.
\begin{DoxyItemize}
\item {\ttfamily data} -\/ directory used to store Boards/\+Library Manager index files and Boards Manager platform installations.
\item {\ttfamily downloads} -\/ directory used to stage downloaded archives during Boards/\+Library Manager installations.
\item {\ttfamily user} -\/ the equivalent of the Arduino IDE\textquotesingle{}s \href{sketch-specification.md\#sketchbook}{\texttt{ "{}sketchbook"{} directory}}. Library Manager installations are made to the {\ttfamily libraries} subdirectory of the user directory.
\end{DoxyItemize}
\item {\ttfamily library} -\/ configuration options relating to Arduino libraries.
\begin{DoxyItemize}
\item {\ttfamily enable\+\_\+unsafe\+\_\+install} -\/ set to {\ttfamily true} to enable the use of the {\ttfamily -\/-\/git-\/url} and {\ttfamily -\/-\/zip-\/file} flags with \`{}arduino-\/cli lib install\`{}. This is the specification for Arduino sketches.

The programs that run on Arduino boards are called \char`\"{}sketches\char`\"{}. This term was inherited from \href{https://processing.org/}{\texttt{ Processing}}, upon which the Arduino IDE and the core API were based.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md41}{}\doxysection{Sketch folders and files}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md41}
The sketch root folder name and code file names must start with a basic letter ({\ttfamily A}-\/{\ttfamily Z} or {\ttfamily a}-\/{\ttfamily z}) or number ({\ttfamily 0}-\/{\ttfamily 9}), followed by basic letters, numbers, underscores ({\ttfamily \+\_\+}), dots ({\ttfamily .}) and dashes ({\ttfamily -\/}). The maximum length is 63 characters.

Support for names starting with a number was added in Arduino IDE 1.\+8.\+4.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md42}{}\doxysubsection{Sketch root folder}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md42}
Because many Arduino sketches only contain a single .ino file, it\textquotesingle{}s easy to think of that file as the sketch. However, it is the folder that is the sketch. The reason is that sketches may consist of multiple code files and the folder is what groups those files into a single program.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md43}{}\doxysubsection{Primary sketch file}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md43}
Every sketch must contain a {\ttfamily .ino} file with a file name matching the sketch root folder name.

{\ttfamily .pde} is also supported but {\bfseries{deprecated}} and will be removed in the future, using the {\ttfamily .ino} extension is strongly recommended.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md44}{}\doxysubsection{Additional code files}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md44}
Sketches may consist of multiple code files.

The following extensions are supported\+:


\begin{DoxyItemize}
\item .ino -\/ \href{https://www.arduino.cc/reference/en/}{\texttt{ Arduino language}} files.
\item .pde -\/ Alternate extension for Arduino language files. This file extension is also used by Processing sketches. .ino is recommended to avoid confusion. $\ast$$\ast${\ttfamily .pde} extension is deprecated and will be removed in the future.$\ast$$\ast$
\item .cpp -\/ C++ files.
\item .c -\/ C Files.
\item .S -\/ Assembly language files.
\item .h -\/ Header files.
\item .tpp, .ipp -\/ Header files (available from Arduino CLI 0.\+19.\+0).
\end{DoxyItemize}

For information about how each of these files and other parts of the sketch are used during compilation, see the Sketch build process documentation.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md45}{}\doxysubsection{$<$tt$>$src$<$/tt$>$ subfolder}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md45}
The contents of the {\ttfamily src} subfolder are compiled recursively. Unlike the code files in the sketch root folder, these files are not shown as tabs in the IDEs.

This is useful for files you don\textquotesingle{}t want to expose to the sketch user via the IDE\textquotesingle{}s interface. It can be used to bundle libraries with the sketch in order to make it a self-\/contained project.

Arduino language files under the {\ttfamily src} folder are not supported.


\begin{DoxyItemize}
\item In Arduino IDE 1.\+6.\+5-\/r5 and older, no recursive compilation was done.
\item In Arduino IDE 1.\+6.\+6 -\/ 1.\+6.\+9, recursive compilation was done of all subfolders of the sketch folder.
\item In Arduino IDE 1.\+6.\+10 and newer, recursive compilation is limited to the {\ttfamily src} subfolder of the sketch folder.
\end{DoxyItemize}\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md46}{}\doxysubsection{$<$tt$>$data$<$/tt$>$ subfolder}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md46}
The {\ttfamily data} folder is used to add additional files to the sketch, which will not be compiled.

Files added to the sketch via the Arduino IDE\textquotesingle{}s {\bfseries{Sketch \texorpdfstring{$>$}{>} Add File...}} are placed in the {\ttfamily data} folder.

The Arduino IDE\textquotesingle{}s {\bfseries{File \texorpdfstring{$>$}{>} Save As...}} only copies the code files in the sketch root folder and the full contents of the {\ttfamily data} folder, so any non-\/code files outside the {\ttfamily data} folder are stripped.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md47}{}\doxysubsection{Metadata}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md47}
Arduino CLI and Arduino Web Editor use a file named sketch.\+json, located in the sketch root folder, to store sketch metadata.

The {\ttfamily cpu} key contains the board configuration information. This can be set via \`{}arduino-\/cli board attach\`{} or by selecting a board in the Arduino Web Editor while the sketch is open. With this configuration set, it is not necessary to specify the {\ttfamily -\/-\/fqbn} or {\ttfamily -\/-\/port} flags to the \`{}arduino-\/cli compile\`{} or \`{}arduino-\/cli upload\`{} commands when compiling or uploading the sketch.

The {\ttfamily included\+\_\+libs} key defines the library versions the Arduino Web Editor uses when the sketch is compiled. This is Arduino Web Editor specific because all versions of all the Library Manager libraries are pre-\/installed in Arduino Web Editor, while only one version of each library may be installed when using the other Arduino development software.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md48}{}\doxysubsection{Secrets}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md48}
Arduino Web Editor has a \href{https://create.arduino.cc/projecthub/Arduino_Genuino/store-your-sensitive-data-safely-when-sharing-a-sketch-e7d0f0}{\texttt{ "{}Secret tab"{} feature}} that makes it easy to share sketches without accidentally exposing sensitive data (e.\+g., passwords or tokens). The Arduino Web Editor automatically generates macros for any identifier in the sketch which starts with {\ttfamily SECRET\+\_\+} and contains all uppercase characters.

When you download a sketch from Arduino Web Editor that contains a Secret tab, the empty {\ttfamily \#define} directives for the secrets are in a file named arduino\+\_\+secrets.\+h, with an {\ttfamily \#include} directive to that file at the top of the primary sketch file. This is hidden when viewing the sketch in Arduino Web Editor.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md49}{}\doxysubsection{Documentation}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md49}
Image and text files in common formats which are present in the sketch root folder are displayed in tabs in the Arduino Web Editor.\hypertarget{md_examples__arduino__r_e_a_d_m_e_autotoc_md50}{}\doxysubsection{Sketch file structure example}\label{md_examples__arduino__r_e_a_d_m_e_autotoc_md50}
